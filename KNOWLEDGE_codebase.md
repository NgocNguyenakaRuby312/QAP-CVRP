# KNOWLEDGE: Codebase Structure & Implementation Guide

**Purpose:** Define the exact file structure, naming conventions, and implementation order for the QAP-DRL codebase.
**When to reference:** Creating new files, understanding project layout, implementing components.
**Last updated:** April 2026 (Phase 2: 4D amplitudes)

---

## 0. Current Architecture (Phase 2: 4D)

- Amplitude: ψ ∈ ℝ⁴ (S³ hypersphere), W ∈ ℝ^{4×5}
- Rotation: 6 Givens angles for SO(4), MLP 5→32→6
- Context: ℝ⁸ = [ψ'_curr(4), cap/C, t/N, x_curr, y_curr], Wq ∈ ℝ^{4×8}
- Scoring: q·ψ'_j + λ·E_kNN(j) − μ·dist(vₜ,vⱼ), μ clamp [0,20], λ clamp [-2,3]
- Critic: 4→64→1
- Total params: ~609 (budget: 618)
- Scheduler: CosineAnnealingWarmRestarts(T_0=50, T_mult=2)
- No weight_decay on μ

## 4. Train Script Pattern

`train_n20.py` is the base. All other sizes import and override settings:
```python
# train_n50.py (30 lines)
import train_n20
train_n20.GRAPH_SIZE = 50
train_n20.CAPACITY = 40
train_n20.BATCH_SIZE = 256
train_n20.KNN_K = 15
train_n20.OUTPUT_DIR = ...
train_n20.main()
```

Module-level constants inherited by all sizes: `AMP_DIM=4`, `HIDDEN_DIM=32`, `ENTROPY_COEF=0.03`, `MU_INIT=0.5`.

## 5. Key Invariants

1. `psi` and `psi_prime` have L2 norm = 1.0 on S³ — QAP mode only
2. `knn_indices` never contain self-loops (diagonal=inf before topk)
3. Depot (index 0) never masked as infeasible
4. Feasibility mask applied BEFORE softmax (set to -1e9)
5. `context_query.forward()` returns a 2-tuple — always unpack
6. `psi_prime` DETACHED before critic head in ppo_agent.update()
7. Feature order: `[d/C, dist_depot, x, y, angle/π]` — 5D
8. At depot: ψ'_curr = zero vector(4D) but x_curr = depot_x, y_curr = depot_y
9. `mu_param` must be `nn.Parameter`, not a plain float
10. Post-training reload must pass `amp_dim=AMP_DIM, hidden_dim=HIDDEN_DIM`
